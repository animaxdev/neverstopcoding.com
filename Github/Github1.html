Working with Git Files and Folders

In this section we are going to talk about how to add and remove files or folders from your github repository.

Adding and Removing files

Once you create a repository, you will want to add and remove files and even folders. The first thing you want to do is figure out which directory you want to use in order to add or remove folders. This can be done by navigating to that directory in the terminal using the cd (change directory) command. Simply type in 'cd <path name>', so cd followed by the directory you want to change into. You can jump back to a previous directory by typing in 'cd ..'.

Code: 
    // Navigate to the directory
    cd Desktop/GitHub_Files

Once you are in the folder which contains the files you want to add to your git repository, you want to add those files or directories. However, before we can do that we need to ensure that github is configured to recognize all the files and folders. We can do this using the 'git init', if the folder is already has a configation folder (.git), then you will just reinitialize the configuation file.

Code: 
    // Create git init file in directory
    git init

Now we can type in 'git add .' to add all the folders and files in the currently directory. Or we can just type in 'git add <file name>' or 'git add <folder name>'. This command just let's git add the file or directory to the queue, we still have to commit and push the file or directory on to our master branch afterwards. If you wanted to check the status of all the files and folders being tracked or pending in the queue we could type in 'git status'.

Code: 
    // add hello.cpp file to repository
    git add hello.cpp
    
    // check the status of our git tracking
    git status

If for example we want to add a different file or remove the file from the queue, then we can simply reset github by typing in 'git reset <path>' or just 'git reset' if you are in the current directory. Type in git status again to see if the file was removed from the queue. However, if you are satisfied with the files added to the queue and you would like to commit them to the master branch. The next step would be to use the 'git commit' command. In order to make the commit command understandable to others on the team or to yourself. You can add a message like so 'git command -m "add this cpp file"'. Once again you can reset if you don't want to commit the file. The two forms of resetting are soft reset and a hard reset. A soft reset will save changes you made prior to trying to commit, while a hard reset will discard any changes made.

Code: 
    // commit files or folders to repository
    git commit -m "message contained in quotes"
    
    // reset the queue to remove possible files from being added or commited
    git reset
    
    // soft reset saves changes made
    git reset HEAD^ --soft
    
    // hard reset discards changes made
    git reset HEAD^ --hard

If you content with all the files or folders you want to add to the repository, you have commited them, the final step is to push them on to the repository. Be sure to reset if there are some files you don't want to add, because you will need to remove them after they have been commited to github servers. Sometimes files that you have locally won't be in sync with the files on the repository, and you will have errors with trying to commit new files or folders. For this, use 'git pull' in order to sort our any discrepancies

Code: 
    // Ensure that files local files are synced with remote repository
    git pull
    
    // Push all of our commited files to our repository
    git push

Let's look at adding a simply file, with all the commands together.

Code: 
    // add a file
    git add hello.cpp
    
    // commit the file
    git commit -m "adding a file"
    
    // push committed file to repo
    git push

The process is the same for removing this same file from our respository. Only we are using the rm command instead of the add command. Also if you are removing a folder then add the '-r' attribute to the command to recursively delete all the files within the folder.

Code: 
    // add a file
    git rm hello.cpp
    
    // commit the file
    git commit -m "removing a file"
    
    // push commit to remove the file
    git push
